# Overview of the yahoofin Flutter Package

The `yahoofin` package is a powerful tool for Flutter developers looking to integrate financial data from Yahoo Finance into their applications. This package allows you to fetch stock market data, historical prices, and other financial information seamlessly. 

## When to Use This Package
- **Financial Applications**: If you're building an app that requires real-time stock prices or historical financial data.
- **Investment Tracking**: For applications that help users track their investments and analyze market trends.
- **Market Analysis Tools**: If you need to create tools for analyzing stock performance or comparing different financial instruments.

## Key Features
- **Real-time Data**: Access to live stock prices and market data.
- **Historical Data**: Fetch historical stock prices for analysis.
- **Easy Integration**: Simple API for fetching data with minimal setup.
- **Cross-Platform Support**: Works on both Android and iOS.

The `yahoofin` package is an excellent choice for developers looking to incorporate financial data into their Flutter applications with ease and efficiency.

# Setting Up the yahoofin Package

To get started with the `yahoofin` package, follow these steps:

## Installation

1. **Add Dependency**: Open your `pubspec.yaml` file and add the `yahoofin` package under dependencies:

   ```yaml
   dependencies:
     flutter:
       sdk: flutter
     yahoofin: ^latest_version
   ```

   Replace `latest_version` with the most recent version of the package.

2. **Install Packages**: Run the following command in your terminal to install the new dependency:

   ```bash
   flutter pub get
   ```

## Platform-Specific Configuration

### Android
- Ensure that your `minSdkVersion` in `android/app/build.gradle` is set to at least 21:

   ```groovy
   android {
       ...
       defaultConfig {
           ...
           minSdkVersion 21
           ...
       }
   }
   ```

### iOS
- For iOS, ensure that you have the correct permissions set in your `Info.plist` file if you plan to access the internet:

   ```xml
   <key>NSAppTransportSecurity</key>
   <dict>
       <key>NSAllowsArbitraryLoads</key>
       <true/>
   </dict>
   ```

# Complete Example of Using yahoofin

```dart
import 'package:flutter/material.dart';
import 'package:yahoofin/yahoofin.dart';

void main() {
  runApp(RealFlutter());
}

class RealFlutter extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: Text('Yahoo Finance Example')),
        body: StockDataWidget(),
      ),
    );
  }
}

class StockDataWidget extends StatefulWidget {
  @override
  _StockDataWidgetState createState() => _StockDataWidgetState();
}

class _StockDataWidgetState extends State<StockDataWidget> {
  String stockPrice = 'Loading...'; // Initial state for stock price

  @override
  void initState() {
    super.initState();
    fetchStockData(); // Fetch stock data when the widget is initialized
  }

  Future<void> fetchStockData() async {
    // Fetching stock data for a specific ticker symbol
    var stock = await YahooFinance.getQuote('AAPL'); // Get quote for AAPL
    setState(() {
      stockPrice = stock.price.toString(); // Update state with fetched price
    });
  }

  @override
  Widget build(BuildContext context) {
    return Center(
      child: Text('AAPL Stock Price: $stockPrice'), // Display stock price
    );
  }
}

// Application Flow Explanation:
// 1. The main function runs the RealFlutter app.
// 2. RealFlutter builds a MaterialApp with a Scaffold containing an AppBar and StockDataWidget.
// 3. StockDataWidget is a StatefulWidget that manages its own state.
// 4. In the initState method, fetchStockData is called to retrieve stock data when the widget is created.
// 5. fetchStockData uses the YahooFinance API to get the current stock price for AAPL.
// 6. Once the data is fetched, setState is called to update the stockPrice variable.
// 7. The build method then displays the updated stock price in the center of the screen.
