# Altibbi Flutter Package: A Comprehensive Overview
=====================================================

The Altibbi Flutter package is a powerful tool for integrating medical consultations and telemedicine services into your Flutter applications. It provides a seamless and efficient way to connect patients with medical professionals, making it an ideal solution for healthcare providers and medical institutions.

### Key Features:

*   Secure and reliable video consultations
*   Real-time chat and messaging
*   Scheduling and appointment management
*   Integration with existing healthcare systems
*   Customizable and user-friendly interface

### When to Use:

*   Telemedicine and remote healthcare services
*   Medical consultation and appointment scheduling
*   Patient engagement and communication
*   Healthcare system integration and optimization

# Setting Up and Using the Altibbi Flutter Package
=============================================

### Step 1: Add the Package to Your Project

Add the following dependency to your `pubspec.yaml` file:

```yml
dependencies:
  altibbi: ^1.0.0
```

Then, run `flutter pub get` to install the package.

### Step 2: Import the Package

Import the package in your Dart file:

```dart
import 'package:altibbi/altibbi.dart';
```

### Step 3: Initialize the Package

Initialize the package with your API key and secret:

```dart
Altibbi.initialize(
  apiKey: 'YOUR_API_KEY',
  apiSecret: 'YOUR_API_SECRET',
);
```

### Step 4: Use the Package Features

Use the package features, such as scheduling a consultation:

```dart
Altibbi.scheduleConsultation(
  patientId: 'PATIENT_ID',
  doctorId: 'DOCTOR_ID',
  consultationType: ConsultationType.video,
);
```

### Platform-Specific Details:

*   For Android, add the following permissions to your `AndroidManifest.xml` file:

    ```xml
<uses-permission android:name="android.permission.CAMERA" />
<uses-permission android:name="android.permission.RECORD_AUDIO" />
```

*   For iOS, add the following permissions to your `Info.plist` file:

    ```xml
<key>NSCameraUsageDescription</key>
<string>Can I use the camera please?</string>
<key>NSMicrophoneUsageDescription</key>
<string>Can I use the microphone please?</string>
```

```dart
import 'package:flutter/material.dart';
import 'package:altibbi/altibbi.dart';

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'Altibbi Demo',
      theme: ThemeData(
        primarySwatch: Colors.blue,
      ),
      home: MyHomePage(),
    );
  }
}

class MyHomePage extends StatefulWidget {
  @override
  _MyHomePageState createState() => _MyHomePageState();
}

class _MyHomePageState extends State<MyHomePage> {
  // Initialize the Altibbi package
  @override
  void initState() {
    super.initState();
    Altibbi.initialize(
      apiKey: 'YOUR_API_KEY',
      apiSecret: 'YOUR_API_SECRET',
    );
  }

  // Schedule a consultation
  void _scheduleConsultation() {
    Altibbi.scheduleConsultation(
      patientId: 'PATIENT_ID',
      doctorId: 'DOCTOR_ID',
      consultationType: ConsultationType.video,
    );
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('Altibbi Demo'),
      ),
      body: Center(
        child: ElevatedButton(
          onPressed: _scheduleConsultation,
          child: Text('Schedule Consultation'),
        ),
      ),
    );
  }
}

// The application flow is as follows:
// 1. The user opens the app and the Altibbi package is initialized with the API key and secret.
// 2. The user clicks the "Schedule Consultation" button, which calls the _scheduleConsultation function.
// 3. The _scheduleConsultation function uses the Altibbi package to schedule a consultation with the specified patient and doctor.
// 4. The consultation is scheduled and the user is notified.

// In summary, this code demonstrates how to use the Altibbi package to schedule a consultation in a Flutter app.
// It initializes the package, schedules a consultation, and handles the consultation flow.
```
