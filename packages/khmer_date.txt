# Khmer Date Flutter Package
The `khmer_date` Flutter package provides a simple way to work with Khmer dates in your Flutter applications. It allows you to convert Gregorian dates to Khmer dates and vice versa, making it a useful tool for developers who need to support Khmer language and culture in their apps.

## Features
- Convert Gregorian dates to Khmer dates
- Convert Khmer dates to Gregorian dates
- Support for both Android and iOS platforms

## When to Use
This package is useful when you need to display dates in Khmer format in your Flutter application, or when you need to work with Khmer dates in your app's logic.

# Setting Up the Khmer Date Package
To use the `khmer_date` package in your Flutter app, follow these steps:

### Step 1: Add the Package to Your Pubspec File
Add the following line to your `pubspec.yaml` file:
```yml
dependencies:
  khmer_date: ^1.0.0
```
Then, run `flutter pub get` in your terminal to get the package.

### Step 2: Import the Package
Import the package in your Dart file:
```dart
import 'package:khmer_date/khmer_date.dart';
```
### Step 3: Use the Package
You can now use the package to convert dates. For example:
```dart
void main() {
  // Create a Gregorian date
  DateTime gregorianDate = DateTime.now();

  // Convert the Gregorian date to a Khmer date
  KhmerDate khmerDate = KhmerDate.fromGregorianDate(gregorianDate);

  // Print the Khmer date
  print(khmerDate.toString());
}
```
### Platform-Specific Details
The `khmer_date` package works on both Android and iOS platforms. No additional setup is required.

<-- START_MAIN -->

```dart
import 'package:flutter/material.dart';
import 'package:khmer_date/khmer_date.dart';

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'Khmer Date Demo',
      theme: ThemeData(
        primarySwatch: Colors.blue,
      ),
      home: MyHomePage(),
    );
  }
}

class MyHomePage extends StatefulWidget {
  @override
  _MyHomePageState createState() => _MyHomePageState();
}

class _MyHomePageState extends State<MyHomePage> {
  DateTime _gregorianDate = DateTime.now();
  KhmerDate _khmerDate;

  @override
  void initState() {
    super.initState();
    _khmerDate = KhmerDate.fromGregorianDate(_gregorianDate);
  }

  void _convertDate() {
    setState(() {
      _khmerDate = KhmerDate.fromGregorianDate(_gregorianDate);
    });
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('Khmer Date Demo'),
      ),
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: <Widget>[
            Text(
              'Gregorian Date:',
              style: TextStyle(fontSize: 24),
            ),
            Text(
              _gregorianDate.toString(),
              style: TextStyle(fontSize: 24),
            ),
            SizedBox(height: 20),
            Text(
              'Khmer Date:',
              style: TextStyle(fontSize: 24),
            ),
            Text(
              _khmerDate.toString(),
              style: TextStyle(fontSize: 24),
            ),
            SizedBox(height: 20),
            ElevatedButton(
              onPressed: _convertDate,
              child: Text('Convert Date'),
            ),
          ],
        ),
      ),
    );
  }
}

// The application flow is as follows:
// 1. The user runs the app and the `MyApp` widget is created.
// 2. The `MyApp` widget builds the `MyHomePage` widget.
// 3. The `MyHomePage` widget initializes the `_gregorianDate` and `_khmerDate` variables.
// 4. The user sees the Gregorian date and the Khmer date displayed on the screen.
// 5. When the user presses the "Convert Date" button, the `_convertDate` function is called.
// 6. The `_convertDate` function updates the `_khmerDate` variable by converting the `_gregorianDate` to a Khmer date.
// 7. The UI is updated to display the new Khmer date.

```
