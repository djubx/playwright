# Overview of the `gdpr_dialog` Flutter Package

The `gdpr_dialog` package is a Flutter plugin designed to help developers comply with the General Data Protection Regulation (GDPR) by providing a customizable dialog that informs users about data collection practices. This package is particularly useful for applications that collect personal data, ensuring that users are aware of their rights and the data being collected.

## When to Use This Package

You should consider using the `gdpr_dialog` package in scenarios such as:
- **User Data Collection**: If your app collects any form of personal data (e.g., email addresses, location data), you need to inform users about this and obtain their consent.
- **Compliance Requirements**: If your app operates in regions where GDPR compliance is mandatory, this package can help you meet legal requirements.
- **User Trust**: Enhancing transparency with users about data usage can build trust and improve user experience.

## Features

- **Customizable Dialog**: The package allows you to customize the dialog's appearance and content to match your app's branding.
- **Consent Management**: It provides functionality to manage user consent, allowing you to track whether users have accepted or declined data collection.
- **Multi-language Support**: The package supports multiple languages, making it easier to reach a broader audience.

# Tutorial: Setting Up and Using the `gdpr_dialog` Package

## Step 1: Adding the Dependency

To use the `gdpr_dialog` package, you need to add it to your `pubspec.yaml` file. Open the file and add the following line under dependencies:

```yaml
dependencies:
  flutter:
    sdk: flutter
  gdpr_dialog: ^1.0.0  # Check for the latest version on pub.dev
```

Run `flutter pub get` to install the package.

## Step 2: Platform-Specific Configuration

### Android Configuration

1. Open `android/app/src/main/AndroidManifest.xml`.
2. Ensure you have the following permissions:

```xml
<uses-permission android:name="android.permission.INTERNET"/>
```

### iOS Configuration

1. Open `ios/Runner/Info.plist`.
2. Add the following key to request permission for data usage:

```xml
<key>NSAppTransportSecurity</key>
<dict>
    <key>NSAllowsArbitraryLoads</key>
    <true/>
</dict>
```

## Step 3: Using the Package

In your Flutter application, you can use the `gdpr_dialog` package as follows:

1. Import the package:

```dart
import 'package:gdpr_dialog/gdpr_dialog.dart';
```

2. Create a function to show the GDPR dialog:

```dart
void showGdprDialog(BuildContext context) {
  GdprDialog(
    title: 'Data Privacy',
    description: 'We collect your data to improve your experience.',
    acceptText: 'Accept',
    declineText: 'Decline',
    onAccept: () {
      // Handle user acceptance
      print('User accepted data collection.');
    },
    onDecline: () {
      // Handle user decline
      print('User declined data collection.');
    },
  ).show(context);
}
```

3. Call this function in your main widget, typically in the `initState` method or when the app starts.

```dart
@override
void initState() {
  super.initState();
  WidgetsBinding.instance.addPostFrameCallback((_) {
    showGdprDialog(context);
  });
}
```

This setup will ensure that the GDPR dialog is displayed to users when they first open the app.

# Complete Example of Using the `gdpr_dialog` Package

```dart
import 'package:flutter/material.dart';
import 'package:gdpr_dialog/gdpr_dialog.dart';

void main() {
  runApp(RealFlutter());
}

class RealFlutter extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'GDPR Dialog Example',
      home: HomeScreen(),
    );
  }
}

class HomeScreen extends StatefulWidget {
  @override
  _HomeScreenState createState() => _HomeScreenState();
}

class _HomeScreenState extends State<HomeScreen> {
  @override
  void initState() {
    super.initState();
    // Show the GDPR dialog when the app starts
    WidgetsBinding.instance.addPostFrameCallback((_) {
      showGdprDialog(context);
    });
  }

  void showGdprDialog(BuildContext context) {
    // Create and show the GDPR dialog
    GdprDialog(
      title: 'Data Privacy',
      description: 'We collect your data to improve your experience.',
      acceptText: 'Accept',
      declineText: 'Decline',
      onAccept: () {
        // Handle user acceptance
        print('User accepted data collection.');
      },
      onDecline: () {
        // Handle user decline
        print('User declined data collection.');
      },
    ).show(context);
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('GDPR Dialog Example'),
      ),
      body: Center(
        child: Text('Welcome to the GDPR Dialog Example!'),
      ),
    );
  }
}

// Application Flow Explanation:
// 1. The app starts with the main function, which runs the RealFlutter widget.
// 2. The RealFlutter widget builds a MaterialApp with a title and a home screen.
// 3. The HomeScreen widget initializes and shows the GDPR dialog in its initState method.
// 4. The showGdprDialog function creates a GdprDialog instance with a title, description, and buttons for acceptance and decline.
// 5. When the user interacts with the dialog, the corresponding callback is triggered, allowing you to handle their choice.
// 6. The main screen displays a welcome message to the user.
```
